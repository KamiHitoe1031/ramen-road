# 🍜 らーめん道 ～至高の一杯～ ゲーム企画書

## コンセプト

**「ドラフトで具材を取り合い、3×3の丼に盛り付けて、味と芸術で勝負する対戦デジタルボードゲーム」**

寿司Go！のドラフトの駆け引き × 枯山水の配置の美学 × ラーメンという最強テーマ。
全員同じお客さんの好みを見ながら、自分のキャラの強みを活かして最高の一杯を作る。

---

## 基本情報

| 項目 | 内容 |
|------|------|
| ジャンル | ドラフト式対戦ボードゲーム |
| プレイ人数 | 3-4人（オンライン対戦） |
| 1プレイ時間 | 5-8分 |
| アートスタイル | ゆるい手描き風・絵本風 |
| 技術スタック | Phaser 3 + Node.js + Socket.io |
| サーバー | Railway |
| 対象 | カジュアルゲーマー・ボドゲ好き |

---

## ゲームの流れ

```
🏠 ロビー
│  部屋作成（3人 or 4人選択）→ ルームコード共有 → 全員参加
│
├─ 👨‍🍳 キャラ選択（15秒）
│    6人のラーメン屋キャラから1人選ぶ（早い者勝ち）
│
├─ 📋 お客さん公開
│    今回のお客さん2人がオープン。全員共通の目標。
│
├─ 🍲 スープ選択（10秒）
│    4種から1つ同時選択（被りOK）
│
├─ 🍜 麺選択（10秒）
│    3種から1つ同時選択（被りOK）
│
├─ 🥩 具材ドラフト（3-4分）
│    手札から1枚選ぶ → 残りを左隣へ → 繰り返し
│    ・3人プレイ: 手札10枚 → 9枚選ぶ（1枚捨て）
│    ・4人プレイ: 手札9枚 → 9枚全部取る
│
├─ 🎨 盛り付けフェーズ（60秒）
│    取った具材を3×3グリッドにドラッグ＆ドロップ
│    9個全部置いてもいいし、余白を残してもいい
│
├─ 📊 採点フェーズ
│    レイヤー1: 基本ルール（固定）
│    レイヤー2: キャラボーナス
│    レイヤー3: お客さん評価
│    レイヤー4: 称号セレモニー
│
└─ 🏆 結果発表 → リマッチ or ロビーへ
```

### レイヤー1：基本ルール（固定・常時公開）

いつでも確認できる普遍的な採点基準。

#### スープ × 麺の相性テーブル

| | 細麺 | ちぢれ麺 | 太麺 |
|--|:--:|:--:|:--:|
| **豚骨** | **+4** | +1 | +0 |
| **醤油** | +2 | **+3** | +1 |
| **味噌** | +0 | **+4** | +2 |
| **塩** | +1 | +1 | **+3** |

#### 彩りボーナス

丼に配置した具材の色タグ（赤・緑・黄・白・茶・黒・ピンク）の種類数で採点。

| 色の種類数 | ボーナス |
|-----------|---------|
| 2色 | +1点 |
| 3色 | +3点 |
| 4色 | +5点 |
| 5色以上 | +8点 |

#### 隣接相性ボーナス

3×3グリッド上で上下左右に隣り合う具材の組み合わせで加点。
（斜めは隣接扱いにしない）

**良い組み合わせ（+2点/ペア）：**

| 具材A | 具材B | 理由 |
|-------|-------|------|
| チャーシュー | ネギ | 王道の相棒 |
| チャーシュー | 煮たまご | ご褒美コンビ |
| バター | コーン | 札幌の鉄板 |
| 海苔 | ほうれん草 | 家系の定番 |
| 紅しょうが | にんにく | 博多のパンチ |
| わかめ | えび | 海鮮ハーモニー |
| 角煮 | 煮たまご | じっくり煮込み |
| もやし | メンマ | シャキシャキコンビ |

**悪い組み合わせ（-1点/ペア）：**

| 具材A | 具材B | 理由 |
|-------|-------|------|
| バター | 紅しょうが | 味がケンカ |
| えび | にんにく | 繊細さが消える |
| わかめ | コーン | 世界観が迷子 |

#### 中央ボーナス

中央マス（5番）に配置された具材に+1点。

#### 同具材ペナルティ

同じ具材を2つ以上配置した場合、2つ目以降は-1点/個。
（ドラフトで同じ具材を複数取った場合のリスク）

#### 2人のお客さんの組み合わせ例

**例1：タケシ（こってり）× サクラ（映え）**
→ 方向性が違う。「量で攻めるか、美しさで攻めるか」の戦略分岐が鮮明。

**例2：ダイスケ（正統派）× レイ（芸術）**
→ どちらも「丁寧に作る」方向。隣接配置の精度が勝負の分かれ目。

**例3：コウタ（ボリューム）× ヨシコ（あっさり）**
→ 完全に矛盾！片方に寄せるか、中間を狙うか悩ましい。

## 3×3グリッドの座標定義

```
┌─────┐ ┌─────┐ ┌─────┐
│(0,0)│ │(1,0)│ │(2,0)│   ← 上段
└─────┘ └─────┘ └─────┘
┌─────┐ ┌─────┐ ┌─────┐
│(0,1)│ │(1,1)│ │(2,1)│   ← 中段（中央=(1,1)）
└─────┘ └─────┘ └─────┘
┌─────┐ ┌─────┐ ┌─────┐
│(0,2)│ │(1,2)│ │(2,2)│   ← 下段
└─────┘ └─────┘ └─────┘
```

**隣接判定：** 上下左右のみ（斜めは含まない）

**左右対称の判定：**
- (0,0)と(2,0)、(0,1)と(2,1)、(0,2)と(2,2) が「両方空き」or「両方埋まり」で対称

## 技術的注意事項

### パフォーマンス
- ターンベースのため60fps維持は容易
- カードアニメーションはTween使用

### ネットワーク
- Socket.ioでイベント駆動通信
- 送信データは最小限（選択したカードIDのみ等）
- サーバー側で全ゲームロジックを処理（チート防止）

### データ保存
- ゲーム中の状態はサーバーのメモリ内で管理
- 戦績などの永続データは Phase 4 以降で検討

### スケール設定
- Phaser.Scale.FIT でレスポンシブ対応
- 最小タッチターゲット: 44×44px（具材カード・グリッドマス）
